# Unstuck Product Vision

## What Is Unstuck?

Unstuck is a decision reframing and impulse interruption platform.

Its purpose is to help users pause, reflect, and make intentional decisions when faced with impulses, procrastination, avoidance, unhealthy habits, or difficult choices.

Unstuck is built around a simple premise: many poor decisions happen quickly, automatically, and in environments designed to reward immediacy. The product creates a small but meaningful pause between trigger and action, giving the user a chance to see the tradeoff more clearly.

## Problem Statement

Modern environments are optimized for immediate rewards.

People often make decisions that maximize short-term comfort while creating long-term costs. These decisions are not always dramatic. They are often small, repeated choices that compound over time.

Examples include:

- Procrastination
- Doom scrolling
- Unhealthy eating
- Unnecessary alcohol consumption
- Avoidance
- Inaction

Unstuck helps users make those tradeoffs visible.

The product does not assume the user is broken, lazy, or lacking discipline. It assumes the user is operating in a high-friction, high-stimulation environment where intention is easy to lose. Unstuck helps restore that intention at the moment it matters.

## What Unstuck Is

- Decision reframing tool
- Impulse interruption tool
- Reflection tool
- Action-initiation tool

Unstuck helps users slow down enough to ask:

- What am I about to do?
- What am I avoiding?
- What will this cost me later?
- What would a more intentional next action look like?

## What Unstuck Is Not

- Therapy
- Mental health diagnosis
- Religious application
- Productivity tracker
- Habit tracker
- Motivational quote app

Unstuck should stay neutral, practical, and professional. It should avoid exaggerated claims, moral judgment, and motivational language that makes the user feel managed or manipulated.

## Core Loop

```text
Trigger
↓
Interruption
↓
Reflection
↓
Decision Reframing
↓
Intentional Action
↓
Outcome
↓
Learning
```

The core loop begins when the user notices a moment of friction, impulse, avoidance, or uncertainty. Unstuck interrupts the automatic path, guides reflection, reframes the decision, and helps the user choose an intentional next action.

Over time, completed sessions should help the user learn patterns:

- Common blockers
- Recurring avoidance loops
- Situations that lead to short-term decisions
- Actions that reliably help them move forward

## Product vs Tool

The forms are not the product.

The forms are the engine.

The product is the system that helps users:

- Interrupt patterns
- Examine tradeoffs
- Make intentional choices
- Take meaningful action

Forms are the current mechanism for creating structured reflection. They should remain lightweight, direct, and useful. But the long-term value of Unstuck comes from the broader system around those forms: timing, context, reminders, session history, insights, and eventually optional intelligent assistance.

## Long-Term Platform Vision

Unstuck is a cross-platform product with a shared backend and multiple client surfaces.

Long-term platform components may include:

- Native iOS app
- Web application
- Shared Supabase backend
- Profiles
- Notifications
- Analytics
- Optional app interruption / Screen Time integrations
- Optional AI assistance

The iOS app is currently the primary platform. The web app is a companion surface that should eventually reach feature parity where it makes sense.

## Product Principles

Unstuck should be:

- Calm
- Practical
- Neutral
- Respectful of user agency
- Focused on tradeoffs, not judgment
- Lightweight enough to use in the moment
- Structured enough to produce useful reflection

Unstuck should avoid becoming:

- A generic productivity dashboard
- A streak-driven habit game
- A therapy substitute
- A motivational content feed
- A surveillance-heavy blocker app

## Open Product Questions

- What is the final form architecture?
- How aggressive should interruptions be?
- What role should notifications play?
- What role should AI play?
- How should analytics be presented?
- How configurable should app interruption features be?

These questions should be answered through product learning, user feedback, and careful evaluation of the core loop. Future development decisions should be measured against whether they improve the user's ability to interrupt automatic patterns and make more intentional decisions.
